# Agent Collaboration Guide

本文记录本仓库中 Codex 与用户协作时的执行判断规则。

它的作用是把 `AGENTS.md` 中的常驻规则展开成可操作说明。真正的最高优先级规则仍以 `AGENTS.md` 为准。

## 基本原则

- 用户确认任务规格后，低风险本地操作默认直接执行。
- 涉及破坏性、线上、系统级、敏感信息或风险不确定的操作，必须主动询问。
- 用户纠正代理行为时，先判断是否需要沉淀到 `AGENTS.md`、skill 或 docs，不要只在聊天中回应。
- 当规则边界不清楚时，主动提出 1-3 个具体问题让用户决策。

## 什么时候自动执行

在任务文档已经确认后，以下操作默认直接执行：

- 读取文件，例如 `Get-Content`
- 搜索代码，例如 `rg`
- 查看工作区状态，例如 `git status`
- 修改当前任务相关的普通源码或文档
- 运行格式化，例如 `gofmt`
- 运行本地测试，例如 `go test`
- 运行本地构建或检查，例如 `go vet`、`go list`
- 运行仓库约定内生成命令，例如 `wire ./testdata/ping-service/cmd/ping-service/export`

这些操作如果因为 sandbox 或工具环境异常失败，应先区分原因：

- 如果是普通命令失败，按错误信息修复或汇报。
- 如果是 sandbox 初始化或权限问题，说明是工具环境限制，而不是操作本身高风险。
- 不要把所有低风险命令一律升级成需要用户确认。

## 什么时候主动询问

以下情况必须先问用户：

- 删除大量文件
- 改写 Git 历史
- 丢弃未确认修改
- 安装或修改系统级依赖
- 发布、部署或推送线上变更
- 请求第三方或线上 API 修改数据
- 修改真实环境数据库
- 处理未脱敏敏感信息
- 操作会越出当前 workspace
- 执行风险或影响范围不确定
- 用户的目标存在多种合理解释，且猜错会造成返工或风险

询问时应具体说明：

- 准备做什么
- 为什么需要确认
- 影响范围是什么
- 是否有替代方案

不要问空泛问题，例如“你想怎么做”。应给出可决策的问题，例如：

```text
是否允许我把这条规则补进 AGENTS.md？它会成为后续所有任务的常驻规则。
```

## 什么时候沉淀文档

以下内容应主动判断是否写入文档：

- 用户纠正了代理行为，并要求以后不要反复提醒
- 用户问的是刚创建或刚修改文档中的遗漏点
- 回答包含可复用命令、平台差异、路径写法或工作流规则
- 当前任务产生了新的约束、风险、回滚方式或验证方法
- 执行过程中发现原方案不合适，需要改变方向

沉淀位置按以下顺序判断：

| 内容类型 | 推荐位置 |
|---|---|
| 所有任务都必须遵守的长期规则 | `AGENTS.md` |
| specs 流程、确认节点、风险分级细节 | `.agents/skills/spec-workflow/SKILL.md` |
| go-srv-kit 架构、Go、Kratos、Wire、生成命令 | `.agents/skills/go-srv-kit/` |
| 某个任务的目标、方案、执行记录 | `docs/<task-name>/spec.md` |
| 可复用使用说明或操作指南 | `docs/*.md` |

如果不确定应写到哪里，应主动说明判断并向用户提出具体问题。

## AGENTS.md 与 Docs 的关系

`AGENTS.md` 只放简短、长期稳定、跨任务生效的规则。

普通 docs 放解释、示例和操作指南。普通 docs 不一定每次都会自动进入上下文，所以关键规则不能只写在 docs 中。

当某条规则需要“以后都生效”时，应至少在 `AGENTS.md` 写一句短规则，再把详细说明放到 docs 或 skill。

## Skill 的作用

skill 用于承载更具体的流程或领域知识。

本仓库常用 skill：

- `.agents/skills/spec-workflow/`：规格先行、确认节点、风险分级、执行记录
- `.agents/skills/go-srv-kit/`：仓库架构、Go 规范、Kratos、Wire、生成命令

当用户任务同时涉及“流程”和“代码实现”时，应同时使用这两个 skill。

## 示例

### 用户问 PowerShell 路径写法

正确处理：

1. 直接回答 PowerShell 中建议给路径加双引号，不需要写 `\\`。
2. 判断这属于 `docs/codex-cli-usage-guide.md` 的补充内容。
3. 更新该文档。
4. 简要告诉用户更新位置。

### 用户指出“为什么不更新 AGENTS.md”

正确处理：

1. 解释 `AGENTS.md` 与 skill/docs 的分层。
2. 判断该规则是否长期、跨任务适用。
3. 如果适用，直接更新 `AGENTS.md`。
4. 如果不适用，说明应放在哪个 skill 或 docs。

### 已确认规格后需要运行 gofmt

正确处理：

1. 直接运行 `gofmt`。
2. 如果失败，按失败原因处理。
3. 不要再次询问用户是否允许格式化。

## 反例

以下行为应避免：

- 只在聊天中回答可复用规则，却不更新刚创建的使用文档。
- 用户已经确认规格后，仍反复询问是否允许读取文件或运行本地测试。
- 用户指出规则缺口后，只补 skill，不判断是否也应补 `AGENTS.md`。
- 遇到边界不清时沉默跳过，而不是提出具体问题。
