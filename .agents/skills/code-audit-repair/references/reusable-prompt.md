# 代码库综合审计、修复和文档补全

本参考用于全库或多模块维护任务。执行前先遵循当前仓库的 `AGENTS.md`、repo-local skills、构建说明和用户确认流程。

## 综合审计提示词

```text
你是一个严谨的代码维护代理，请对当前代码库执行一次“测试覆盖、功能正确性、安全/性能风险、能力缺口和使用文档”综合审计与修复。

基本要求：

1. 先读取项目根目录的协作规则和构建说明，例如 AGENTS.md、README.md、CONTRIBUTING.md、Makefile、go.mod/package.json/pyproject.toml/Cargo.toml 等。遵循项目既有风格、架构边界、命名和测试习惯。
2. 枚举所有包含源码文件的目录或模块。按语言和项目结构识别哪些是工具库、业务模块、生成代码、示例代码、测试数据、第三方代码。
3. 对每个目标目录检查是否已有测试文件。已有测试时运行并判断是否覆盖核心功能；没有测试或覆盖不足时补充最小有效测试。
4. 运行测试验证功能正确性。不要只写测试不运行；最终必须明确列出实际运行过的命令和结果。
5. 阅读代码时同步检查：
   - 功能缺口：常见场景是否缺失，例如缺少 context/cancellation、常用 HTTP method、可配置 client/timeout、输入校验、错误返回、跨平台处理等。
   - 安全风险：路径穿越、弱随机、硬编码密钥、敏感信息日志、TLS 跳过校验、命令注入、SQL/模板注入、反序列化风险、越权或不可信输入。
   - 稳定性风险：nil panic、数组/切片越界、类型断言 panic、非必要 panic、死循环、goroutine 泄漏、资源未关闭、半成品文件残留。
   - 性能风险：大文件一次性读入内存、无界缓存、热路径反射、过大 buffer、重复编译正则、测试里真实长 sleep。
   - API 兼容性：优先兼容新增，不破坏现有调用；如必须破坏接口，先记录原因、影响和迁移方式。
6. 对能低风险修复的问题，直接做最小必要修改，并补充回归测试。对高风险或需要设计取舍的问题，记录为后续建议，不要擅自大改。
7. 为每个工具/模块补充本目录 README.md，内容保持短小：
   - 用途
   - 基础用法
   - 注意事项
   - 验证命令
   如果已有 README.md，只补充缺失内容，不要覆盖有价值的现有说明。
8. 在 docs/ 下建立或更新总索引文档，只保留通用约定和目录链接。不要把所有模块说明写成一个巨大文档。
9. 生成一份审计清单文档，列出：
   - 每个目录发现的问题
   - 已处理的修复和测试
   - 已扫描但暂不改代码的原因
   - 后续建议和风险
10. 修改后执行格式化、静态检查或测试命令。优先跑受影响模块的定向测试，再跑全量测试。如果因环境问题无法执行，说明具体命令、错误和替代验证；不要伪造测试结果。

工作方式：

- 先读代码和相邻实现，再决定改法。
- 优先做最小兼容改动。
- 不手改生成文件。
- 不回退用户已有改动。
- 不做无关重构。
- 不安装系统依赖、不发布、不部署、不改写 Git 历史。
- 遇到破坏性操作、线上操作、敏感信息或高风险设计变更时先停止并说明。

最终输出：

- 简要说明实际完成了什么。
- 列出主要修改目录和新增/修改文件。
- 列出实际运行过的验证命令和结果。
- 指出未处理的风险、暂不改原因和后续建议。
```

## 审计清单

- 测试：是否有测试、是否能跑、是否覆盖正常路径、错误路径、边界输入、并发和资源释放。
- 输入：检查 nil、空字符串、负数、零值、超大值、非法格式、IPv6/IPv4、路径分隔符和跨平台差异。
- 错误：是否保留底层错误、是否包含足够上下文、是否吞错、是否覆盖错误路径测试。
- Context：网络、命令、下载、异步任务是否支持取消和超时。
- IO：文件是否 close、失败是否留下半成品、是否自动创建受控目录、是否防路径穿越。
- 网络：timeout、client 注入、TLS 默认安全、状态码处理、响应体关闭。
- 随机：安全场景使用 `crypto/rand` 或等价安全随机源；非安全随机要在文档说明。
- 加解密：检查 key 长度、IV/nonce、padding 校验、伪造密文错误路径、密钥不进日志。
- 并发：检查 goroutine recover、context 传播、锁释放，测试避免长时间 sleep。
- 性能：大文件流式处理，避免热路径反射和无界内存增长。
- 文档：每个目录有本地 README；总文档只做索引；文档示例必须和真实 API 对齐。

## 输出文件建议

- 任务过程记录写入当前仓库约定的临时任务文档。
- 长期可复用规则写入 `AGENTS.md`、repo-local skill、模块 README 或稳定 docs。
- 模块说明写入对应模块的 `README.md`。
- 总索引只保留稳定文档和目录链接，不引用临时任务文档。
