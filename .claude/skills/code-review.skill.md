# Code Review 技能

## 描述
按照项目规范进行代码审查，检查代码质量、安全性、性能和最佳实践。

## 审查维度
1. **代码规范**：检查是否符合 coding-standards.md
2. **架构合规**：检查是否遵循 DDD 分层架构
3. **错误处理**：检查是否正确使用 kratos/error 包
4. **安全性**：检查 SQL 注入、XSS、敏感信息等
5. **性能**：检查 N+1 查询、缓存使用等
6. **测试**：检查测试覆盖率和测试质量

## 审查输出格式
```markdown
## Code Review 报告

### ✅ 通过的检查
- 项目结构符合 DDD 分层
- 错误处理正确

### 🔴 必须修复
1. **文件**: `internal/service/service/user.service.go:25`
   - **问题**: Service 层直接调用 Data 层
   - **建议**: 应该通过 Business 层调用

### 💡 优化建议
1. **文件**: `internal/data/data/user.data.go:50`
   - **当前**: 使用循环查询单个用户
   - **建议**: 使用批量查询减少数据库调用

### 📊 测试覆盖率
- 当前: 45%
- 目标: 70%
```
